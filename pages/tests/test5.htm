Тестирование 5. Технология управления данными об изделии (PDM)

1. Основная идея PDM:
Упростить процесс создания руководств
Повысить эффективность управления информацией об изделии
Создание электронного архива чертежей

2. Основные задачи, решаемые PDM:
Создание ЕИП; автоматизация управления создания изделия; управление производством изделия; хранение стандартов и чертежей
Создание архива документации; создание электронного макета изделия; оптимизация процессов создания
Создание ЕИП; автоматизация управления конфигурацией; построение системы качества; создание электронного архива чертежей и документации
3. Пользователи PDM-систем:
Конструкторы изделия
Все сотрудники предприятия
Технологи изделия

4. С чем необходима интеграция PDM-систем?
С ПО конструкторов
С ПО отдела кадров
Со всем ПО, которое используется в ЖЦ изделия

5. Хранение данных должно обеспечивать:
Целостность данных, контроль доступа, поиск, архивирование
Поиск данных, контроль доступа, чтение данных
Архивирование, поиск, контроль доступа, конвертация

6. Управление с помощью PDM-систем должно обеспечивать:
Отслеживание правильности работ, определение сроков создания изделия, определение стоимости создания изделия
Управление работой, управление потоками работ, протоколирование работы
Выбор конкретного исполнителя, выбор контролирующего процесс специалиста

7. Управление составом изделия должно обеспечить:
Поддержку информации об изделии, поддержку разных конфигураций изделия, оценку влияния изменения компонента изделия, применяемость компонента изделия
Изменение изделия, определение разных конфигураций, хранение конфигураций
Хранение данных об изделии в определённой структуре

8. Уровни интеграции:
Общая система команд, общие запросы к БД, общий формат данных
Единая модель данных, прямой доступ к БД, прикладные программные интерфейсы, конверторы
Унификация всех команд и форматов используемых ПО в ЖЦ изделия

9. Что такое Smart Flow?
Cредство описания выполнения и контроля бизнесс-процессов
Cредство для обмена данными о составе изделия
Cредство защищенной или открытой "публикации" информации на WEB

10. Сервер защищенного хранилища данных (файлов произвольных форматов) - это
Smart Gateway
Smart Vault
Smart BOM

11. Описание метаданных в Team PDM организовано в виде:
Дерева объектов
Классической структуры иерархических классов
Перевернутого дерева"

К классу коллаборативных систем
К классу комбинированных систем
К классу самодостаточных систем

Поиск по атрибутам
